KQL Reference

A quick-reference of KQL operators, functions, and ready-to-use query patterns for study and day-to-day SOC work. Click Copy on any block and paste it into the Practice tab (or your real Sentinel / advanced-hunting workspace).

Query basics

A KQL query starts with a table and pipes (|) data through a sequence of operators. Each operator transforms the rows. Comments use //.

TableName | where SomeColumn == "value" // filter rows | project ColumnA, ColumnB // choose columns | sort by TimeGenerated desc // order | take 10 // first 10 rows

Filter & shape rows

OperatorWhat it does
whereKeep rows matching a condition
projectChoose / rename columns (drops the rest)
project-awayRemove specific columns
extendAdd a new calculated column
distinctUnique rows for the named columns
take / limitReturn N arbitrary rows
sort by / order byOrder rows (default desc)
top N by CHighest N rows by a value
countNumber of rows (column Count)
SigninLogs | where ResultType != "0" | extend Failed = true | project TimeGenerated, UserPrincipalName, IPAddress, Failed | sort by TimeGenerated desc

Aggregate with summarize

summarize groups rows and computes aggregates. Group with by. Bucket time with bin().

AggregationResult
count()Row count (default column count_)
countif(pred)Count rows where a condition is true
dcount(Col)Distinct count of a column
sum / avg / min / max (Col)Numeric/temporal aggregates
make_set(Col) / make_list(Col)Collect values into an array (unique / all)
SigninLogs | summarize Total = count(), Failures = countif(ResultType != "0"), Countries = make_set(Location) by UserPrincipalName | sort by Failures desc

Combine tables

union stacks rows from multiple tables; join merges rows on a matching key. Join kinds: inner, innerunique, leftouter, leftanti.

// union: rows from both tables together union SigninLogs, AuditLogs | where TimeGenerated > ago(1d) // join: correlate on a shared key SecurityAlert | where Severity == "High" | project UserPrincipalName = CompromisedEntity | join kind=inner (SigninLogs) on UserPrincipalName | distinct UserPrincipalName

Working with time

// last 24 hours SigninLogs | where TimeGenerated > ago(1d) // a specific window SigninLogs | where TimeGenerated between (datetime(2026-06-20) .. datetime(2026-06-21)) // events per hour (for a timechart) SigninLogs | summarize count() by bin(TimeGenerated, 1h) | sort by TimeGenerated asc | render timechart

Timespans: s seconds, m minutes, h hours, d days — e.g. ago(30m), bin(TimeGenerated, 15m).

Basic queries you'll reuse

These run against the Practice database (and mirror real Sentinel tables).

// 1) Recent activity SigninLogs | where TimeGenerated > ago(24h)
// 2) Failed sign-ins only SigninLogs | where ResultType != "0"
// 3) Count events by a field SigninLogs | summarize count() by ResultType
// 4) Top talkers SigninLogs | summarize Count = count() by UserPrincipalName | top 5 by Count desc
// 5) Distinct values SigninLogs | distinct Location
// 6) Failed Windows logons (Event 4625) SecurityEvent | where EventID == 4625 | summarize count() by Account, Computer

Advanced hunting patterns

// Brute-force: many failures from one user/IP SigninLogs | where ResultType != "0" | summarize Failures = count() by UserPrincipalName, IPAddress | where Failures >= 5 | sort by Failures desc
// How many distinct hosts did each account touch? SecurityEvent | where EventID == 4624 | summarize Hosts = dcount(Computer), HostList = make_set(Computer) by Account | sort by Hosts desc
// Suspicious PowerShell (encoded commands) DeviceProcessEvents | where FileName == "powershell.exe" and ProcessCommandLine has_any ("-enc", "-EncodedCommand") | project TimeGenerated, DeviceName, AccountName, ProcessCommandLine
// Rare processes across the fleet (potential LOLBins) DeviceProcessEvents | summarize Count = count(), Devices = dcount(DeviceName) by FileName | sort by Count asc
// Correlate: users with a High alert who also signed in SecurityAlert | where Severity == "High" | project UserPrincipalName = CompromisedEntity | join kind=inner (SigninLogs) on UserPrincipalName | distinct UserPrincipalName
// Activity timeline for a single user SigninLogs | where UserPrincipalName == "bpatel@contoso.com" | summarize Attempts = count() by bin(TimeGenerated, 1h), ResultType | sort by TimeGenerated asc

Note: has_any and render are shown for real-world reference; the Practice sandbox supports a core subset (it accepts render and most patterns above run there).

More techniques worth knowing

General-purpose KQL you'll use constantly. Some (marked) are beyond the Practice sandbox but standard in Sentinel / Azure Data Explorer.

// let: reusable variables and sub-queries let lookback = 7d; let vips = dynamic(["ceo@contoso.com","cfo@contoso.com"]); SigninLogs | where TimeGenerated > ago(lookback) | where UserPrincipalName in (vips)
// case / iff: conditional (calculated) columns SigninLogs | extend Outcome = iff(ResultType == "0", "Success", "Failure") | extend Bucket = case(ResultType == "0", "ok", ResultType == "50126", "bad-password", "other")
// extract fields from text with parse / extract SecurityEvent | parse Account with Domain "\\" User | project Account, Domain, User
// first-seen / last-seen per entity with arg_min / arg_max DeviceProcessEvents | summarize FirstSeen = arg_min(TimeGenerated, ProcessCommandLine), LastSeen = max(TimeGenerated) by DeviceName, FileName
// mv-expand: one row per array element SigninLogs | summarize Ips = make_set(IPAddress) by UserPrincipalName | mv-expand Ips
// anti-join: things in A that are NOT in B (e.g. new hosts today) DeviceInfo | where TimeGenerated > ago(1d) | join kind=leftanti (DeviceInfo | where TimeGenerated between (ago(30d) .. ago(1d))) on DeviceName | distinct DeviceName // hosts seen today but never in the prior 30 days
// time-series with make-series + render (Sentinel/ADX) SigninLogs | make-series Count = count() default = 0 on TimeGenerated from ago(1d) to now() step 1h by ResultType | render timechart
// percentiles & richer aggregates DeviceNetworkEvents | summarize Total = count(), Uniq = dcount(RemoteIP), p95 = percentile(RemotePort, 95) by DeviceName

SOC hunting library

Job-relevant hunts grouped by data source. Adapt table/column names to your workspace.

Identity & sign-ins

// Impossible travel: same user, 2+ countries in a short window SigninLogs | where ResultType == "0" | summarize Countries = make_set(Location), Logins = count() by UserPrincipalName, bin(TimeGenerated, 1h) | where array_length(Countries) > 1
// Password spray: one IP, many distinct users failing SigninLogs | where ResultType != "0" | summarize Users = dcount(UserPrincipalName), Attempts = count() by IPAddress | where Users >= 5 | sort by Users desc
// MFA fatigue: repeated MFA prompts for a user SigninLogs | where ResultType == "50074" // MFA required | summarize Prompts = count() by UserPrincipalName, bin(TimeGenerated, 15m) | where Prompts >= 5

Endpoint

// LOLBins spawning from Office apps DeviceProcessEvents | where InitiatingProcessFileName in ("winword.exe","excel.exe","outlook.exe") | where FileName in ("powershell.exe","cmd.exe","wscript.exe","mshta.exe","rundll32.exe") | project TimeGenerated, DeviceName, InitiatingProcessFileName, FileName, ProcessCommandLine
// Beaconing: repeated connections to the same destination DeviceNetworkEvents | summarize Hits = count(), Ports = make_set(RemotePort) by DeviceName, RemoteIP | where Hits >= 3 | sort by Hits desc

Email

// Delivered phishing that users then clicked EmailEvents | where ThreatTypes has "Phish" and DeliveryAction == "Delivered" | project NetworkMessageId, RecipientEmailAddress, Subject | join kind=inner (UrlClickEvents | where ActionType == "ClickAllowed") on NetworkMessageId | project RecipientEmailAddress, Subject, Url

Cloud & audit

// Risky principals performing destructive operations AzureActivity | where OperationNameValue has_any ("delete","write") | summarize Ops = count(), Kinds = make_set(OperationNameValue) by Caller, CallerIpAddress | sort by Ops desc
// Privileged group changes AuditLogs | where OperationName has "member" and OperationName has "group" | where TargetResource has_any ("Domain Admins","Global Administrator") | project TimeGenerated, InitiatedBy, OperationName, TargetResource, Result
