SC-200 Cheat Sheet

The highest-yield facts on one page. Skim this the morning of your exam.

Defender XDR — the workloads

  • MDE = Defender for Endpoint (EDR).
  • MDO = Defender for Office 365 (email).
  • MDI = Defender for Identity (on-prem AD).
  • MDA = Defender for Cloud Apps (CASB).
  • Entra ID Protection = cloud identity risk.
  • Managed at security.microsoft.com. Alerts → incidents.

Defender for Endpoint

  • Device actions: Isolate, restrict app execution, run AV scan, collect investigation package, live response.
  • ASR rules = block risky behaviours (audit/block).
  • Custom indicators = allow/block hash, IP, URL, cert.
  • Defender Vulnerability Management = risk-based weaknesses.
  • Onboard via script, GPO, Intune, ConfigMgr.

Defender for Office 365

  • Safe Attachments = sandbox detonation.
  • Safe Links = time-of-click URL scan.
  • ZAP = remove already-delivered bad mail.
  • Tenant Allow/Block List = manual allow/block.
  • Preset policies = Standard/Strict baselines.
  • Threat Explorer = investigate & soft-delete mail.

Identity: MDI vs Entra ID Protection

  • Defender for Identity = on-prem AD; sensors on DCs / AD FS / AD CS.
  • Detects Pass-the-Hash/Ticket, Golden Ticket, DCSync, recon, lateral movement.
  • Entra ID Protection = cloud risky sign-ins/users → risk-based Conditional Access.

Defender for Cloud Apps (CASB)

  • Cloud Discovery = find Shadow IT from logs.
  • App connectors = API deep visibility.
  • Conditional Access App Control = real-time session policies (block download / read-only).
  • Anomaly policies = e.g., impossible travel.

Advanced hunting (KQL)

  • Key tables: DeviceProcessEvents, EmailEvents, IdentityLogonEvents, CloudAppEvents.
  • AlertInfo + AlertEvidence = link alerts to entities.
  • Save a hunting query as a custom detection rule → scheduled alerts + response.

Defender for Cloud — CSPM

  • CNAPP = CSPM (posture) + CWPP (workload).
  • Foundational CSPM is free → recommendations + Secure Score.
  • MCSB = default benchmark driving score.
  • Multicloud: connect AWS/GCP accounts.

Defender for Cloud — plans

  • Defender for Servers → MDE, JIT, FIM, vuln assessment.
  • JIT VM access = open RDP/SSH only on request.
  • FIM = watch critical files/registry.
  • Other plans: Storage, SQL, Containers, App Service, Key Vault, DNS, Resource Manager, APIs.
  • Workflow automation = Logic App on alert.
  • Regulatory compliance dashboard needs a paid plan for extra standards.

Sentinel — basics & roles

  • SIEM + SOAR on a Log Analytics workspace; uses KQL.
  • Roles: Reader (view), Responder (manage incidents), Contributor (content), Playbook Operator (run), Automation Contributor (let Sentinel run playbooks).
  • Commitment tier = reserved daily GB discount.

Sentinel — connectors

  • Data connectors ingest sources (MS first-party often free).
  • CEF/Syslog via Azure Monitor Agent on a Linux forwarder.
  • DCR = defines what AMA collects (control cost).
  • Content hub = packaged solutions.

Sentinel — analytics rules

  • Scheduled = KQL on an interval.
  • NRT = ~every minute (fastest).
  • Microsoft Security = incidents from MS product alerts.
  • Fusion = ML multistage attacks.
  • TI = match logs to indicators. Entity mapping for investigation.

Sentinel — SOAR, hunting, TI

  • Playbook = Logic App workflow (the actions).
  • Automation rule = orchestrate incidents (run playbook, assign, status).
  • Hunting queries (MITRE ATT&CK) → bookmarks.
  • UEBA = behaviour baselines. Watchlist = reference list.
  • TI via TAXII / Upload Indicators → ThreatIntelligenceIndicator.

KQL essentials

  • where = filter · project = pick cols · extend = add col.
  • summarize count() by X = aggregate.
  • join = merge on key · union = stack tables.
  • ago(1d) = relative time · bin() = bucket time.
  • top N by C = highest · take = arbitrary N.

⚠️ Common exam traps

  • On-prem AD → Defender for Identity; cloud identity → Entra ID Protection.
  • Incidents from MS alerts → Microsoft Security rule; multistage ML → Fusion; fastest → NRT.
  • Do the action → playbook; orchestrate the incident → automation rule.
  • Manage incidents but not edit content → Responder (not Reader).
  • Reduce RDP brute force → JIT VM access.
  • Shadow IT → Cloud Discovery; real-time session control → Conditional Access App Control.
  • Free posture + score → foundational CSPM; threat alerts → paid Defender plan.
