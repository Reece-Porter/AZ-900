AZ-900 Cheat Sheet

The highest-yield facts on one page. Skim this the morning of your exam.

Cloud service types — responsibility

  • IaaS = most control; you manage the OS + apps (e.g. Azure VMs).
  • PaaS = you manage apps & data only (App Service, Azure SQL DB).
  • SaaS = least control; just use it (Microsoft 365).
  • Always customer: data, devices, accounts/identities.
  • Always provider: physical hosts, network, datacenter.

Cloud models

  • Public = multi-tenant, over internet, no CapEx.
  • Private = one org, more control, needs CapEx.
  • Hybrid = public + private; move/keep workloads.
  • Multi-cloud = more than one provider.
  • Azure Arc = manage on-prem/multi-cloud from Azure.
  • Azure VMware Solution = run VMware natively in Azure.

Cost: CapEx vs OpEx & benefits

  • CapEx = big upfront purchase (servers).
  • OpEx = pay-as-you-go; cloud favours this.
  • Consumption model = pay only for what you use.
  • Benefits: HA, scalability, elasticity, reliability, predictability, security, governance, manageability.

Scaling & reliability

  • Vertical (up/down) = more power to one resource.
  • Horizontal (out/in) = more/fewer instances.
  • Elasticity = auto scale to demand.
  • High availability = minimal downtime (SLA %).
  • Reliability = recover from failure.

Architecture hierarchy & DR

  • Top→down: Management groups → Subscriptions → Resource groups → Resources.
  • Region = datacenters in a low-latency area.
  • Availability Zones = separate datacenters within a region (datacenter-fault protection).
  • Region pairs = 300+ mi apart, cross-region DR.
  • ARM = deployment/management layer (templates, RBAC, tags, locks).

Compute

  • VMs = IaaS, full OS control, lift-and-shift.
  • VM Scale Sets = identical autoscaling VMs.
  • Availability Sets = fault/update domains in a datacenter.
  • App Service = PaaS web apps/APIs.
  • ACI = single container fast; AKS = orchestrate many.
  • Functions = serverless, event-driven, pay-per-run.
  • Azure Virtual Desktop = cloud Windows desktops.

Networking

  • VNet = isolated private network; subnets divide it.
  • VNet peering = connect two VNets privately.
  • VPN Gateway = encrypted over the internet.
  • ExpressRoute = private, dedicated, not over internet.
  • Azure DNS = name resolution. NSG = traffic rules.
  • Private endpoint = keep storage/PaaS off public internet.

Storage — tiers & redundancy

  • Types: Blob (objects), Files (SMB), Queue, Table, Disk.
  • Tiers: Hot → Cool (30d) → Cold (90d) → Archive (offline, cheapest).
  • LRS = 3 copies, 1 datacenter (cheapest).
  • ZRS = across zones (datacenter-fault).
  • GRS = secondary region (regional outage).
  • GZRS = zones + region (max durability).
  • Move data: Migrate, Data Box (~80TB offline), AzCopy, File Sync.

Identity & security

  • Microsoft Entra ID = cloud identity (SSO).
  • AuthN = who you are; AuthZ = what you can do.
  • MFA = 2+ factors; SSO = sign in once.
  • Conditional Access = rules by user/location/device/risk.
  • B2B = partners/guests; B2C = customers.
  • Entra ID (cloud, modern) vs AD DS (on-prem, Kerberos/LDAP).

Cost management tools

  • Pricing Calculator = estimate cost before deploy.
  • TCO Calculator = on-prem vs Azure savings.
  • Cost Management = track actual spend, budgets, alerts.
  • Tags = name/value metadata for cost grouping.
  • Cost factors: region, resource type/size, egress data, subscription type. (Ingress is free.)

Governance & compliance

  • Azure Policy = what resources can exist/config (allowed regions, required tags).
  • RBAC = who can do what (least privilege).
  • Locks: CanNotDelete (edit, no delete) · ReadOnly (no edit/delete).
  • Microsoft Purview = data governance.
  • Service Trust Portal = compliance reports/certs.
  • Defender for Cloud = security posture + threat alerts.

Tools & monitoring

  • Portal = GUI · Cloud Shell = browser Bash/PowerShell.
  • CLI (az) vs PowerShell (cmdlets) · ARM/Bicep = IaC.
  • Advisor = recommendations (cost/security/reliability/perf/ops).
  • Service Health = is Azure itself up? (outages/maintenance).
  • Azure Monitor = telemetry from YOUR resources.
  • Log Analytics = KQL queries · App Insights = app performance.

⚠️ Common exam traps

  • Datacenter fault → Availability Zones; regional fault → Region pairs/GRS.
  • Estimate before deploy → Pricing; compare on-prem → TCO.
  • Prevent delete but allow edit → CanNotDelete lock (not ReadOnly).
  • Recommendations → Advisor; outages → Service Health.
  • Enforce config/tags → Policy; grant access → RBAC.
  • Cheapest archival w/ slow retrieval → Archive tier.
  • No OS management → PaaS; full OS control → IaaS.
