☁️ Domain 1 — Describe Cloud Concepts 25–30%

What is cloud computing?

Cloud computing is the delivery of computing services — including servers, storage, databases, networking, software, analytics, and intelligence — over the internet (“the cloud”). Instead of buying and maintaining physical hardware, you rent capacity from a provider like Microsoft Azure and pay for what you use.

Key idea: The cloud turns IT from something you build and own into something you consume as a service.

The consumption-based model (CapEx vs OpEx)

Traditional on-premises IT is a capital expense (CapEx): you spend a large amount upfront on hardware that depreciates over time. The cloud uses an operational expense (OpEx), pay-as-you-go model: you pay only for resources you use, when you use them.

CapEx (on-premises)OpEx (cloud)
Large upfront purchase of hardwareNo upfront cost — pay per use
You maintain & depreciate assetsProvider maintains infrastructure
Pay whether used or notStop paying when you stop using
Remember: Consumption-based pricing means no upfront cost, you only pay for what you use, and you can stop at any time.

Benefits of cloud computing

  • High availability: Services stay up with minimal downtime, backed by SLAs (e.g., 99.9%).
  • Scalability: Add resources to meet demand — vertically (scale up/down = more power per resource) or horizontally (scale out/in = more instances).
  • Elasticity: Automatically add/remove resources as demand changes, so you only pay for what's needed.
  • Reliability: The ability to recover from failures, thanks to a decentralized, globally distributed design.
  • Predictability: Cost predictability (forecast spend) and performance predictability (autoscaling, load balancing).
  • Security: Cloud providers offer a large set of policies, technologies, and controls; you choose the level you need.
  • Governance & manageability: Standards and templates keep deployments compliant; manage via portal, CLI, APIs, and automatic scaling.

The shared responsibility model

Security and management responsibilities are shared between you and the cloud provider. Who is responsible for what depends on the service type (IaaS/PaaS/SaaS).

🏗️ Domain 2 — Describe Azure Architecture & Services 35–40%

Core architectural components

Regions, geographies & datacenters
  • Datacenter: A physical facility with servers, networking, power, and cooling.
  • Region: A set of datacenters deployed within a latency-defined perimeter, connected by a low-latency network. You deploy resources into a region.
  • Geography: An area of the world containing at least one region; respects data-residency and compliance boundaries.
  • Region pairs: Two regions in the same geography ≥ 300 miles apart. Updates roll out one region at a time; one region per pair is prioritized in recovery — used for disaster recovery.
  • Sovereign regions: Isolated instances for compliance (e.g., Azure Government, Azure operated by 21Vianet in China).
Availability Zones

Physically separate datacenters within a region, each with independent power, cooling, and networking (minimum 3 in an enabled region). Use them to protect against datacenter-level failures. Pair zones with zone-redundant services for high availability.

Management hierarchy

From top to bottom: Management groups → Subscriptions → Resource groups → Resources.

  • Resource: A manageable item (VM, database, storage account, VNet).
  • Resource group: A logical container for related resources. A resource lives in exactly one group; deleting the group deletes everything inside.
  • Subscription: A unit of billing and access management. Acts as a boundary for limits/quotas.
  • Management group: Containers above subscriptions for applying policy and access across many subscriptions at once.
Azure Resource Manager (ARM)

ARM is the deployment and management layer. Every request (portal, CLI, PowerShell, SDK, REST) goes through ARM, which authenticates and authorizes it. ARM enables templates (IaC), RBAC, tags, and locks.

Containers vs VMs

VMs virtualize hardware and each runs its own full OS (heavier, slower to start). Containers virtualize the OS, are lightweight, start in seconds, and share the host kernel — ideal for microservices and portability.

Networking services

  • Virtual Network (VNet): An isolated private network in Azure; resources communicate securely with each other, the internet, and on-premises. Divided into subnets.
  • VNet peering: Connects two VNets privately over the Microsoft backbone.
  • VPN Gateway: Encrypted connection over the public internet — site-to-site (network↔network) or point-to-site (client↔network).
  • ExpressRoute: Private, dedicated connection that does not traverse the public internet — higher speed, reliability, security.
  • Azure DNS: Hosting service for DNS domains using Azure infrastructure for name resolution.
VPN vs ExpressRoute: VPN = encrypted over the internet (cheaper). ExpressRoute = private link that bypasses the internet (faster, more secure, more expensive).

Migration & file movement tools
  • Azure Migrate: Central hub to discover, assess, and migrate servers, databases, and apps.
  • Azure Data Box: Physical appliance for offline transfer of large datasets (~80 TB).
  • AzCopy: Command-line tool to copy blobs/files to/from storage.
  • Azure Storage Explorer: GUI app to manage storage data.
  • Azure File Sync: Sync Azure Files with a local Windows Server cache.

Identity, access & security

  • Microsoft Entra ID (formerly Azure AD): Cloud identity & access management — handles sign-in and access to apps/resources.
  • Authentication (AuthN): Proving who you are. Authorization (AuthZ): Determining what you can access. AuthN first.
  • Single Sign-On (SSO): Sign in once, access many apps.
  • Multi-Factor Authentication (MFA): Two+ factors — something you know / have / are.
  • Conditional Access: Grant/block/step-up (MFA) access based on signals (user, location, device, risk).
  • External identities: B2B (partners/guests use their own credentials) and B2C (customer/consumer identities).

🛡️ Domain 3 — Describe Azure Management & Governance 30–35%

Cost management

Factors that affect cost
  • Resource type and its settings/size.
  • Services consumed (each has its own pricing).
  • Location/region — prices vary by region.
  • Data transfer — ingress (inbound) is usually free; egress (outbound) is charged.
  • Subscription type (e.g., free trial vs pay-as-you-go) and Azure Marketplace purchases.
Cost tools
ToolPurpose
Pricing CalculatorEstimate cost of services before deploying
TCO CalculatorCompare on-premises vs Azure costs (savings)
Microsoft Cost ManagementMonitor, analyze & optimize actual spend; budgets & alerts
TagsGroup resources to track and report costs
Easy mix-up: Pricing Calculator = before deployment (estimate). Cost Management = after deployment (track actual). TCO = on-prem vs cloud comparison.

Governance & compliance

  • Azure Policy: Create/assign rules so resources stay compliant (e.g., allowed regions, required tags, allowed VM sizes). Can audit or enforce.
  • Role-Based Access Control (RBAC): Assign roles (permissions) scoped to management group, subscription, resource group, or resource. Least-privilege, allow-based.
  • Resource locks: Prevent accidental changes — CanNotDelete (can read/modify, not delete) and ReadOnly (read only).
  • Microsoft Purview: Unified data governance & compliance across on-prem, multi-cloud, and SaaS data.
  • Service Trust Portal: Microsoft's published audit reports, certifications, and compliance resources.
Policy vs RBAC: RBAC controls who can do what (access). Azure Policy controls what resources can exist / how they're configured (compliance). Locks protect against accidental delete/change.

Tools for managing & deploying

  • Azure portal: Web GUI to build, manage, and monitor resources.
  • Azure Cloud Shell: Browser-based Bash or PowerShell — no local install.
  • Azure CLI: Cross-platform az commands.
  • Azure PowerShell: Cmdlets (Verb-Noun) for scripting management.
  • Azure Arc: Manage on-prem, multi-cloud, and edge resources from Azure.
  • ARM templates / Bicep: Infrastructure as code — declarative, repeatable deployments. Bicep is a cleaner language that compiles to ARM JSON.

🎯 Exam-day tips

  • Read every question fully — watch for words like NOT, always, and least/most.
  • Eliminate obviously wrong answers first; many questions hinge on one keyword.
  • There's no penalty for wrong answers — never leave a question blank.
  • “Yes/No” case questions can't be revisited once you move on — answer carefully.
  • Manage time: ~45 minutes for 40–60 questions ≈ under a minute each. Flag and return.
